# MP 新增 Stage 内 Media 约束测试用例（基于 PRD）

## 1. 来源与目标

- PRD：`MP新增stage 约束`
- 目标：验证“不同 stage 下 media 占比波动较大”问题是否被约束机制有效收敛，并保证在不可行场景下有可解释兜底行为。

## 2. 验收规则（从 PRD 抽取）

1. 作用域：对**每个推广区域**和**全局汇总**都生效。
2. 当 `stage`、`media` 优先级都生效时：
  - 最小值：stage 内每个存在的 media 满足 `actual_ratio >= min(10%, target_media_ratio / 2)`；
  - 容差：
    - `media_match_type=完全匹配`：先用用户容差；不可行时放宽到 `max(25%, 用户容差)`；
    - `media_match_type=大小关系匹配`：stage 内使用默认容差 `15%`，同时媒体总和保持大小关系。
3. 当 `stage`、`media` 有一个不生效时：仅校验最小值，不强制 stage 内容差。
4. 某 stage 缺失某 media：跳过该 stage 的该 media 容差检查，仅保留最小值。

## 3. 通用前置条件

- 数据至少包含 3 个 stage（如 `preheat/open-sale/sustain`）。
- media 至少 3 个（建议 `Meta/Google/TikTok`），且包含“高占比 + 低占比”组合。
- 至少准备 1 个单国家样本（建议 `SG`）和 1 个多国家样本（如 `SG+MY+TH`）。
- 结果输出同时检查：
  - 区域维度；
  - 全局维度。

## 5. 每条用例执行步骤（统一）

1. 构造/选择对应请求参数（match_type、range、priority、国家与stage结构）。
2. 运行 MP 分配，生成结果文件。
3. 运行 KPI/预算检查脚本，产出 `achievement_check.json`。
4. 生成并查看 Markdown/HTML 报告。
5. 按“最小值、容差、排序、作用域、是否触发松弛”五项做判定。

## 6. 统一判定标准

- P0（必须通过）：
  - TC01/TC03/TC05/TC10/TC11/TC12/TC14/TC15；
  - 不允许出现 stage 内 media 低于最小值；
  - 不允许出现“应松弛未松弛”或“无需松弛却错误放宽”。
- P1（应通过）：
  - 其余功能与稳健性用例；
  - 容差、排序、区域/全局口径一致。

## 7. 建议落地到自动化参数模板

- `stage_match_type`: `"完全匹配"` / `"大小关系匹配"`
- `stage_range_match`: `10/20/30`
- `media_match_type`: `"完全匹配"` / `"大小关系匹配"`
- `media_range_match`: `3/5/10/25/30`
- `module_priority_list`: 覆盖 `["stage","media",...]`、`["stage",...]`、`["media",...]`、都不生效四类
- `allow_zero_budget`: `False`（主路径），`True`（回归）
