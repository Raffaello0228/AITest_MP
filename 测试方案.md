# MP 新增 Stage 内 Media 约束测试方案

## 1. 文档信息

- 测试对象：MP 智能分配（Stage 内 Media 预算约束）
- 需求来源：飞书文档《MP新增stage 约束》
- 参考结构：`brief_template_structure.md`
- 目标版本：`20260326_v1`（可按实际版本替换）

## 2. 背景与测试目标

当前 MP 分配结果存在「stage 内 media 预算波动不自然」问题。新增约束后，需要验证：

- 在用户配置 stage 和 media 都生效时，stage 内 media 分配更加稳定；
- 在部分模块不生效时，系统仍能保证基础可解释性（最小分配比例）；
- 在复杂场景（大小关系匹配、stage 缺失 media、严格容差不可行）下有明确兜底行为。

## 3. 规则范围与验收口径

### 3.1 作用域

- 规则对 **每个推广区域** 生效；
- 同时对 **全局汇总** 生效。

### 3.2 核心约束

1. 当 `stage`、`media` 优先级都生效时：
  - 最小值约束：stage 内所有实际存在 media 满足  
   `actual_ratio >= min(10%, target_media_ratio / 2)`
  - 容差约束：
    - 若 `media_match_type=完全匹配`：优先使用用户容差 `media_range_match`；
    - 若不可行：放宽到 `max(25%, media_range_match)`；
    - 若 `media_match_type=大小关系匹配`：stage 内采用默认容差 `15%`，并保持 media 总体大小关系。
2. 当 `stage`、`media` 有一个不生效时：
  - 只要求最小值约束，不强制 stage 内容差约束。
3. 当某个 stage 不存在某 media（例如该 stage 无 FB）：
  - 跳过该 stage 的该 media 容差校验；
  - 仅保留最小值约束。

### 3.3 统一验收指标

- **最小值通过率**：所有有效 stage-media 组合满足下限；
- **容差通过率**：适用容差规则的组合满足区间；
- **排序一致性**：大小关系匹配时，stage 汇总与全局排序一致；
- **可行性兜底**：严格配置不可行时，存在松弛后可行解（符合最大 25% 兜底）；
- **区域/全局一致性**：区域与全局都输出并通过对应检查。

## 4. 测试数据准备

- 选择至少 1 个国家级别样本（建议 SG）+ 1 个多国家样本；
- 覆盖至少 3 个 stage；
- media 至少包含 2 个大媒体（如 Meta、Google）和 1 个小媒体（如 TikTok 子平台）；
- 准备 2 类特殊数据：
  - `stage` 缺失某 media 的 case；
  - 严格容差下不可行、需要触发松弛的 case。

## 5. 用例设计

### 5.1 主用例矩阵（建议 12 条）

1. 新增约束-双生效-完全匹配-容差5
  - 目的：验证标准配置下最小值 + 容差同时满足。
2. 新增约束-双生效-完全匹配-严格容差5
  - 目的：验证严格容差下求解稳定性。
3. 新增约束-双生效-完全匹配-宽容差25
  - 目的：验证宽容差下波动是否收敛到合理范围。
4. 新增约束-双生效-media大小关系-stage内15容差
  - 目的：验证 media 大小关系匹配时，stage 默认 15% 行为。
5. 新增约束-仅stage生效-仅最小值
  - 目的：验证 stage 生效、media 弱化时只保底。
6. 新增约束-仅media生效-仅最小值
  - 目的：验证 media 生效、stage 弱化时只保底。
7. 新增约束-双大小关系匹配
  - 目的：验证 stage+media 同时大小关系下的稳定性。
8. 新增约束-保底边界-大媒体10下限
  - 目的：验证 `target=80%` 时下限为 `min(10,40)=10%`。
9. 新增约束-保底边界-小媒体2点5下限
  - 目的：验证 `target=5%` 时下限为 `min(10,2.5)=2.5%`。
10. 新增约束-区域与全局同时满足
  - 目的：验证两层作用域同时生效。
11. 新增约束-缺失media-跳过容差仅保底
  - 目的：验证 stage 缺失 media 的例外逻辑。
12. 新增约束-不可行时容差松弛到max25
  - 目的：验证“先用户容差、失败后兜底 25%”策略。

### 5.2 建议参数模板（与现有测试字段保持一致）

- `stage_match_type`: `"完全匹配"` / `"大小关系匹配"`
- `stage_range_match`: `10/20/30/None`
- `media_match_type`: `"完全匹配"` / `"大小关系匹配"`
- `media_range_match`: `3/5/10/25/None`
- `module_priority_list`: 至少覆盖 `["stage", "media", ...]` 与仅单模块优先生效两类排列
- `allow_zero_budget`: `False`（主场景），可补充 `True` 回归

## 6. 执行步骤

1. 在 `testcase_templatex_xiaomi_stage_media_stability.py` 增加/维护上述用例；
2. 生成请求并执行 MP 分配；
3. 运行检查脚本，输出 `achievement_check.json`；
4. 生成 HTML/Markdown 报告；
5. 对每条用例按“最小值/容差/排序/作用域”逐项判定；
6. 汇总失败模式并分类：数据问题、约束冲突、算法实现偏差。

## 7. 结果判定标准

- P0（必须通过）：
  - 双生效场景下最小值与容差约束均通过；
  - 缺失 media 逻辑正确（不误报容差失败）；
  - 严格容差不可行时可触发松弛并得到可行结果。
- P1（应通过）：
  - 大小关系场景下排序一致且结果稳定；
  - 区域与全局结果一致性满足预期。
- 不通过判定：
  - 出现 stage 内 media 低于最小值；
  - 应用容差规则错误（未松弛或过度松弛）；
  - 作用域遗漏（只校验了区域或只校验了全局）。

## 8. 风险与关注点

- 测试数据若过于“理想”，不易触发松弛逻辑，建议人为构造冲突 case；
- stage 缺失 media 需要在原始 brief 明确建模，否则无法覆盖例外分支；
- 多国家汇总口径需与单国家口径一致，避免统计口径差异导致误判。

## 9. 交付物
