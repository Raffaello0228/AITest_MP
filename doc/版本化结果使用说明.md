# 算法版本化结果使用说明

系统支持按**算法版本**保留每次评测结果，便于算法更新后快速对比质量。

## 目录约定

- **请求**（可复用，不按版本）：`output/<common|xiaomi>/requests/`
- **结果**（按版本）：`output/<common|xiaomi>/results/<algo_version>/`
- **KPI 检查 JSON**（按版本）：`output/<common|xiaomi>/achievement_checks/json/<algo_version>/`
- **报告**（按版本）：`output/<common|xiaomi>/achievement_checks/reports/<algo_version>/`

同一套 `requests` 可多次跑不同算法版本，结果互不覆盖。

## 版本号来源（优先级从高到低）

1. 命令行参数 `--algo-version`
2. 环境变量 `MEDIAPLAN_ALGO_VERSION`
3. 配置文件 `config/version_config.ini` 中 `[version] algo_version`
4. 默认：`latest`

## 推荐流程

### 1. 首次或算法更新前跑一版（留基线）

```bash
# 指定版本号，例如按日期
set MEDIAPLAN_ALGO_VERSION=20250226_baseline
python tools/batch_execute_requests.py --dir output/common/requests
python tools/batch_check_and_report.py --algo-version 20250226_baseline
python tools/summarize_all_results.py --algo-version 20250226_baseline
```

或直接用命令行：

```bash
python tools/batch_execute_requests.py --dir output/common/requests --algo-version 20250226_baseline
python tools/batch_check_and_report.py --algo-version 20250226_baseline
python tools/summarize_all_results.py --algo-version 20250226_baseline
```

### 2. 算法更新后再跑一版

```bash
python tools/batch_execute_requests.py --dir output/common/requests --algo-version 20250226_v2
python tools/batch_check_and_report.py --algo-version 20250226_v2
python tools/summarize_all_results.py --algo-version 20250226_v2
```

### 3. 快速评测（与上一版对比）

- **版本预算对比报告**（推荐）：对比两个算法版本下各用例、各维度的预算差异，并标出差异较大的项。
  ```bash
  python tools/compare_version_budgets.py --version-a 20250226_baseline --version-b 20250226_v2
  ```
  报告输出到 `output/<common|xiaomi>/achievement_checks/reports/version_compare_<版本A>_vs_<版本B>.html`。  
  可指定 `--pct-threshold`（相对变化%阈值，默认 10）、`--amount-pct-threshold`（绝对差异占总量%阈值，默认 5）控制“差异大”的判定。
- 查看各版本汇总报告：
  - `output/common/achievement_checks/reports/20250226_baseline/summary_all_results.html`
  - `output/common/achievement_checks/reports/20250226_v2/summary_all_results.html`
- 或直接对比两个版本下的 `*_achievement_check.json` 与汇总 HTML。

### 4. 使用“当前”版本（不保留多版本）

不指定 `--algo-version` 时使用配置默认值（一般为 `latest`），结果写入 `results/latest/`、`achievement_checks/json/latest/` 等，每次运行会覆盖该版本。

## 兼容旧结构

- 若结果仍在**无版本子目录**的旧路径（如 `output/common/results/*.json`），做 KPI 检查时可**显式指定输出目录**，或先将结果拷贝到 `output/common/results/latest/` 再跑 `batch_check_and_report.py --algo-version latest`。
- 汇总时可直接指定输入目录：  
  `python tools/summarize_all_results.py --input-dir output/common/achievement_checks/json`  
  用于汇总未按版本分目录的旧结果。
